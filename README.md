# coffee_beans_analyze.py 参数说明文档

## 1. 全局常量

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `IMAGE_SIZE` | 第 6 行 | `3500` | 图像标准化尺寸（像素）。所有轮廓坐标均在此尺寸的坐标系下 |
| `COLOR_THRESHOLD` | 第 7 行 | `210` | 深色像素判断阈值。灰度值 < 210 认为是咖啡豆，≥ 210 认为是背景/空白区域 |

---

## 2. 短轴计算 — `calculate_short_axis_robust`

对每个轮廓做两次判断，防止弧形/薄片轮廓导致 `fitEllipse` 短轴虚高。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| 校正比例 | 第 147 行 | `1.5` | 若 fitEllipse 短轴 > 内切圆直径 × 1.5，认为估算有误，改用内切圆直径。调小（如 1.2）会更激进地修正，调大（如 2.0）则仅在差异极大时才修正 |

---

## 3. 距离峰值切割 — `_split_by_distance_peaks`

适用于近圆形多豆聚簇（shape_ratio > 0.85），通过距离变换找到各豆核心区域后切割轮廓。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `min_area` | 第 321 行函数签名 | `7000` px² | 切割后每一半的最小面积。低于此值则放弃此切割方案 |
| `min_sr` | 第 321 行函数签名 | `0.55` | 切割后每一半的最小形状系数（面积 / 凸包面积）。过低说明切出了奇形轮廓 |
| 高值区域比例 | 第 353 行 | `0.40` | 距离变换 > max × 0.40 的区域被认为是豆的核心。调大（如 0.60）要求核心更集中，更容易分离多峰；调小（如 0.25）则会合并本应分离的峰 |

---

## 4. conv_ratio 颈部切割 — `_neck_split_forced`

通过 `conv_ratio = Dmat / D2`（欧氏距离 / 弧长距离）找到轮廓最窄颈部并切割。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `min_sr` | 第 402 行函数签名 | `0.55` | 切割后每一半的最小形状系数 |
| `min_area` | 第 402 行函数签名 | `7000` px² | 切割后每一半的最小面积 |
| 弧长下限 `n//6` | 第 426 行 | `n//6` | 两切割点之间最少要跨越的轮廓点数。每颗豆约占总轮廓 1/3，故设为 1/6 以保证跨越真正的颈部。调小会更容易触发切割，但可能切到轮廓表面的细小凹陷 |

---

## 5. 颈宽阈值切割 — `_force_split_mesh31`

用户建议的方案：直接以最窄处欧氏距离（颈宽）为判断标准进行切割。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `max_neck_px` | 调用处第 617 行 | `4.0 mm` | 颈宽上限。两颗粘连豆的接触宽度通常 < 4 mm，超过此值不认为是颈部 |
| `min_sr` | 第 442 行函数签名 | `0.40` | 切割后每一半的最小形状系数（比前两个宽松，切割后一半仍可能是多豆聚簇，允许后续迭代继续拆） |
| `min_area` | 第 442 行函数签名 | `7000` px² | 切割后每一半的最小面积 |
| `top_k` | 第 442 行函数签名 | `30` | 最多尝试多少个候选切割点对（按颈宽从小到大排序） |
| 弧长下限 `n//20` | 第 464 行 | `n//20` | 比 `_neck_split_forced` 更宽松，支持 C 形聚簇中弧长较短的颈部 |

---

## 6. 第一遍后处理 — `postprocess_contours` 第一遍

处理 `is_split_granular` 检测到的明显粘连（颈形轮廓）。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| 最大迭代次数 | 第 506 行 `while stack` | 无限（用 stack 驱动） | 每个粘连 blob 都会被递归地继续拆分，直到所有子块不再触发分裂条件 |

---

## 7. 第二遍后处理 — `postprocess_contours` 第二遍

处理面积偏大或形状系数偏低的 blob（侧面相切、桥接形）。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| 面积倍数阈值 | 第 539 行 | `1.5` | 面积 > 中位值 × 1.5 时触发拆分 |
| 形状系数下限 | 第 547 行 | `0.60` | 面积 ≥ 7000 且 shape_ratio < 0.60 时触发拆分（明显凹陷） |
| 距离峰值切割触发门槛 | 第 557 行 | `0.85` | shape_ratio > 0.85 才尝试 `_split_by_distance_peaks`（近圆形聚簇专用） |
| 丢弃门槛 | 第 564 行 | `0.50` | shape_ratio < 0.50 的轮廓直接丢弃（奇形残片，不是咖啡豆） |
| 最大迭代次数 | 第 540 行 | `6` | 防止无限循环 |

---

## 8. 长细比检查 — `postprocess_contours` 第三遍

处理端对端粘连（两颗豆首尾相接）。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| 长宽比阈值 | 第 582 行 | `2.5` | fitEllipse 长轴 / 短轴 ≥ 2.5 时认为是端对端粘连，强制切割 |

---

## 9. mesh=31 强制拆分 — `postprocess_contours` 第四遍

短轴超过 mesh=31 对应尺寸的 blob 必然是粘连豆，三级优先方案强制拆分。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| mesh=31 短轴阈值 | 第 594 行 | `11.91 mm` | 标准筛网 mesh=31 对应的孔径，超过此短轴即为粘连 |
| 颈宽阈值（第二优先） | 第 617 行 | `4.0 mm` | `_force_split_mesh31` 的颈宽上限 |
| 距离峰值触发门槛（第三优先） | 第 621 行 | `0.85` | shape_ratio > 0.85 才尝试 `_split_by_distance_peaks` |
| 最大迭代次数 | 第 596 行 | `6` | 防止无限循环 |

---

## 10. 腐蚀分割法 — `_erode_split`

前三种切割方法均失败时的最终兜底（第四优先）。对轮廓内深色像素掩膜逐步腐蚀，直到连通区域分裂，用各分裂部分的质心指导轮廓切割。适用于 C 形聚簇等复杂粘连形态。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `min_sr` | 函数签名 | `0.40` | 切割后每一半的最小形状系数 |
| `min_area` | 函数签名 | `7000` px² | 切割后每一半的最小面积 |
| 腐蚀步长 | 函数内 | `2` px | 每轮腐蚀半径增量。步长越小精度越高，但速度稍慢 |
| 最大腐蚀半径 | 函数内 | `100` px | 超过此半径仍未分裂则放弃。豆子半径约 80-100px，设为 100 足够 |
| 噪点面积阈值 | 函数内 | `300` px² | 腐蚀后连通域面积 < 300px² 视为噪点，忽略不计 |

---

## 11. 填充率修正短轴 — `granular_recon` 第 8 步

`RETR_EXTERNAL` 轮廓会将 C 形聚簇中间的空白围入，导致 shoelace 面积和 fitEllipse 短轴虚高，目数误判为 31。本步骤对含大量空白的 blob 改用深色像素的内切圆直径。

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| 填充率阈值 | 第 721 行 | `0.88` | `深色像素数 / shoelace面积` < 0.88 时认为轮廓内含大量空白，触发修正。调大（如 0.92）会更积极地修正，调小（如 0.80）则只修正空洞极大的情形 |
| 最小面积 | 第 704 行 | `5000` px² | 面积过小的 blob 跳过填充率检查 |

---

## 参数调优速查

| 问题现象 | 建议调整 |
|----------|----------|
| 漏检豆子（豆太浅被当成背景） | 调大 `COLOR_THRESHOLD`（如 220） |
| 误检背景为豆子 | 调小 `COLOR_THRESHOLD`（如 200） |
| 粘连豆未被拆开 | 调大颈宽阈值 `4.0 mm`，或调小 `_neck_split_forced` 的 `n//6` |
| 正常豆被错误切割 | 调大 `min_sr`（如 0.60），或调大 `n//6`（如 `n//4`） |
| C 形/空洞聚簇目数仍偏大 | 调大填充率阈值（如 0.92） |
| 形状奇怪的残片残留 | 调大丢弃门槛（如 0.55） |
